# 时间胶囊 · 未来开启机制设计

## 背景
目前时间胶囊在封存后即可直接查看原信与 AI 回信，缺少「到未来才能打开」的仪式感与核心玩法。本设计引入真实的 `openAt` 开启时间，只有到达该时间后才能查看内容，并通过「演示模式」让测试者能在几分钟内完整体验流程。

## 目标
1. 让「时间胶囊」名副其实：未到 `openAt` 无法查看信件内容。
2. 提供本地「演示模式」，方便演示与测试。
3. 在列表、详情页清晰表达胶囊状态（封印中 / 可开启 / 已开启）。
4. 开启过程有仪式感动画，增强情感体验。

## 核心规则

### 开启条件
- 每个胶囊包含 `openAt: string`（ISO 8601）。
- 仅当 `new Date() >= new Date(openAt)` 时，胶囊为「可开启」状态。
- 用户点击「开启胶囊」按钮后，`status` 变为 `"opened"`，原信与 AI 回信才可见。
- 已开启胶囊可反复查看内容。

### 状态流转
```
封存(sealed) → 可开启(openable) → 已开启(opened)
```
- `sealed`：已封存，未到开启时间。
- `openable`：不是显式字段，由 `openAt <= now && status !== "opened"` 计算得出。
- `opened`：用户已手动开启。

### 演示模式
- 在 `/write` 页增加「演示模式」开关，状态持久化到 `localStorage`。
- 开启后，新胶囊的 `openAt = now + 3 分钟`。
- 关闭后，新胶囊恢复真实时间（6 个月 / 1 年 / 3 年 / 5 年）。
- 只影响新建胶囊，不影响已有胶囊。

## 数据模型变更

### `TimeCapsuleLetter`
已存在字段 `openAt: string` 保持不变。

### `LetterStatus`
已包含 `"sealed" | "replied" | "opened"`，无需新增。

### 新增工具函数（storage.ts）
- `isLetterOpenable(letter): boolean` — 判断是否到达开启时间且未开启。
- `openLetter(id): void` — 将 `status` 改为 `"opened"` 并通知 UI。
- `calculateOpenDate(recipientTime, demoMode?): Date` — 增加可选 `demoMode` 参数。
- `DEMO_MODE_KEY` / `isDemoMode()` / `setDemoMode(bool)` — 演示模式本地持久化。

## UI 改动

### 1. `/write` 写信页
- 在「收件人时间」选择器附近增加「演示模式」开关。
- 开关文案：「演示模式：3 分钟后即可开启」。
- 开启时表单底部给出小提示：「当前为演示模式，新胶囊将在 3 分钟后可开启。」

### 2. `/capsules` 列表页
- 每个胶囊卡片右上角增加状态标签：
  - 封印中：显示倒计时（天/时/分/秒自动刷新）
  - 可开启：显示「时间已到」
  - 已开启：显示「已阅」
- 状态标签用品牌色区分。

### 3. `/capsules/[id]` 详情页
- **未到开启时间**：
  - 中央显示大型倒计时。
  - 显示「这封信将在 XXXX 年 XX 月 XX 日 XX:XX 开启」。
  - 显示封印插画/火漆印章图标，不展示原信与 AI 回信。
- **已可开启但未点击开启**：
  - 显示「开启胶囊」按钮，带脉冲动画吸引点击。
  - 点击后播放信纸展开动画，随后显示原信与 AI 回信。
- **已开启**：
  - 直接显示原信与 AI 回信（保持现有布局）。

### 4. 新增组件（可选）
- `CountdownDisplay`：通用倒计时组件，用于列表与详情。
- `SealLock`：封印状态视觉组件，替代原信内容占位。

## 兼容性
- 旧数据可能没有 `openAt`：视为已可开启，显示「开启胶囊」按钮。
- 已有 `aiReply` 但未开启的胶囊：AI 回信同样封存，到时间后一并显示。

## 错误处理
- `openLetter` 找不到 ID 时静默返回。
- 倒计时组件在组件卸载时清理定时器。

## 验收标准
- [ ] 真实时间下新胶囊的 `openAt` 正确计算。
- [ ] 演示模式下新胶囊 3 分钟后可开启。
- [ ] 列表页状态标签随时间自动更新。
- [ ] 详情页未到时间不显示原信与 AI 回信。
- [ ] 点击「开启胶囊」后内容可见，且刷新后仍保持已开启。
- [ ] 构建无错误，移动端正常显示。
